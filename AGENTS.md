# obsidian-second-brain — Fork Operating Guide

> Written for AI agents working with this fork.
> This file describes how this fork differs from upstream and why.

---

## Purpose

This is a fork of [eugeniughelbur/obsidian-second-brain](https://github.com/eugeniughelbur/obsidian-second-brain) — a cross-CLI skill that turns an Obsidian vault into an AI-maintained second brain.

Our goal is to make the project **platform-neutral** and add first-class **OMP** support, so agents on any CLI (OMP, Claude Code, Codex CLI, Gemini CLI, OpenCode) can use the same skill without confusion.

## Workflow for maintaining this fork

After pulling from upstream:

```bash
git pull upstream main
bash scripts/convert.sh --setup    # renames + assume-unchanged
bash scripts/build.sh --platform omp  # rebuild OMP dist
```

---

## Maintenance rules

- **`scripts/convert.sh`** is the single source of truth for all text replacements. Add new patterns there, not inline.
- **Files tagged `assume-unchanged`** should never be committed in converted form. Only our fork's own files (`scripts/convert.sh`, `adapters/omp/adapter.sh`, `FORK_*.md`, etc.) are committed.
- **Ephemeral transform outputs** (renamed templates, etc.) are `.gitignore`'d.
- **There are no CI pipelines or formatters** — just the four files above.
- **This file (`AGENTS.md`)** should be maintained and extended by any agent that finds something worth preserving for future agent work. Add sections, conventions, or gotchas as they arise.
